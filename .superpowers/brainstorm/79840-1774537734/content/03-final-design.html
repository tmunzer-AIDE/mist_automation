Design summary:
  • Split view — Chat panel (left) + Data panel (right, ~340px)
  • Chat narration — AI messages with timestamps, system event dividers, user Q&A
  • Time-based progress — Smooth bar based on monitoring_started_at → monitoring_ends_at, updated every second via setInterval
  • Verdict banner — Appears on completion between header and split view
  • No-LLM mode — Same layout, timeline entries as system messages, no input bar
  • User input — Uses existing LLM globalChat/followUp with session context passed as page_context
  • Reuses — Existing AiChatPanel patterns (markdown, streaming, tool calls), LlmService, WebSocket channels
